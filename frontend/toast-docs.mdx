#### Toast Lifecycle Rules

Toast dismissal behavior depends on the presence of actions and duration:

- **Toast with no action**

  - Auto-dismisses after `duration` or **5000ms (default)**.

- **Toast with `actionText` or `onAction`**

  - Does **not auto-dismiss** by default.
  - Stays visible until:
    - Action is clicked, or
    - User manually closes it.

- **Toast with action + explicit `duration`**
  - Auto-dismisses after the specified duration.

These rules prevent users from missing important actionable messages.

This documentation outlines the implementation and usage of a toast notification system in your Next.js project.

The system supports **automatic dismissal**, **action-based persistence**, and
**explicit duration control**, making it suitable for success messages,
errors, and interactive notifications.

#### Toast Object Structure

The `showToast` function accepts the following object:

| Field         | Type                              | Required | Description                               |
| ------------- | --------------------------------- | -------- | ----------------------------------------- |
| `type`        | `"success" \| "error" \| "draft"` | Yes      | Visual style of the toast.                |
| `title`       | `string`                          | Yes      | Short heading for the toast.              |
| `description` | `string`                          | Yes      | Detailed message content.                 |
| `actionText`  | `string`                          | No       | Label for optional action button.         |
| `onAction`    | `() => void`                      | No       | Callback executed when action is clicked. |
| `duration`    | `number`                          | No       | Auto-dismiss time in milliseconds.        |

> `id` is generated internally and should not be provided.

#### Toast Component

The `Toast` component displays a list of notifications with different types (success, error, draft). Each toast includes a title, description, and optional action.

**Code Snippet:**

```tsx
import React from "react";
import { useToast } from "./toast-context";

const Toast = () => {
  const { toasts, removeToast } = useToast();

  return (
    <div className="fixed bottom-4 right-4 space-y-4 z-50">
      {toasts.map(({ id, type, title, description, actionText, onAction }) => (
        <div
          key={id}
          className={`flex items-start p-4 rounded-md shadow-lg w-[300px] bg-[#262626]`}
        >
          {/* Icon */}
          <div className="mt-1">
            {type === "success" && <SuccessIcon />}
            {type === "error" && <ErrorIcon />}
            {type === "draft" && <DraftIcon />}
          </div>

          {/* Content */}
          <div className="ml-3 flex-1">
            <h3 className={`text-sm font-semibold ${getColor(type)}`}>
              {title}
            </h3>
            <p className={`text-xs mt-1 ${getDescriptionColor(type)}`}>
              {description}
            </p>

            {actionText && (
              <button
                onClick={() => {
                  if (onAction) onAction();
                  removeToast(id);
                }}
                className={getActionButtonStyle(type)}
              >
                {actionText}
              </button>
            )}
          </div>

          <button
            onClick={() => removeToast(id)}
            className={`ml-4 hover:text-gray-300 ${getCloseButtonStyle(type)}`}
          >
            ✕
          </button>
        </div>
      ))}
    </div>
  );
};

export default Toast;
```

#### Toast Context

The `toast-context` file manages toast state, lifecycle timing,
and dismissal rules using a centralized provider.

#### Behavioral Notes

- Toasts are removed using timers managed by `useEffect`.
- Multiple toasts can coexist simultaneously.
- Toast IDs are generated using `Date.now()`.
- Removing a toast clears its dismissal timer automatically.
- Action-based toasts are intentionally persistent unless dismissed.

**Code Snippet:**

```tsx
import { createContext, useContext, useState } from "react";

const ToastContext = createContext(undefined);

export const ToastProvider = ({ children }) => {
  const [toasts, setToasts] = useState([]);

  const showToast = (toast) => {
    setToasts([...toasts, { ...toast, id: Date.now() }]);
  };

  const removeToast = (id) => {
    setToasts(toasts.filter((toast) => toast.id !== id));
  };

  return (
    <ToastContext.Provider value={{ toasts, showToast, removeToast }}>
      {children}
    </ToastContext.Provider>
  );
};

export const useToast = () => {
  const context = useContext(ToastContext);
  if (!context) throw new Error("useToast must be used within a ToastProvider");
  return context;
};
```

#### Toast Layout

The `ToastLayout` component combines the `ToastProvider` and the `Toast` component. Wrap your app with this layout to enable toast notifications.

**Code Snippet:**

```tsx
import { ToastProvider } from "./toast-context";
import Toast from "./Toast";

const ToastLayout = ({ children }) => {
  return (
    <ToastProvider>
      {children}
      <Toast />
    </ToastProvider>
  );
};

export default ToastLayout;
```

### Usage

1. Wrap your application with the `ToastLayout` component:

```tsx
import ToastLayout from "./ToastLayout";

const MyApp = ({ Component, pageProps }) => {
  return (
    <ToastLayout>
      <Component {...pageProps} />
    </ToastLayout>
  );
};

export default MyApp;
```

2. Use the `useToast` hook to display a toast notification:

```tsx
import { useToast } from "./toast-context";

const MyComponent = () => {
  const { showToast } = useToast();

  const handleClick = () => {
    showToast({
      type: "success",
      title: "Success!",
      description: "Your operation was successful.",
    });
  };

  return <button onClick={handleClick}>Show Toast</button>;
};

export default MyComponent;
```

```tsx
showToast({
  type: "draft",
  title: "Draft Saved",
  description: "Your work has been saved as a draft.",
  actionText: "View",
  onAction: () => console.log("Viewing draft"),
  duration: 3000,
});
```

### Customization

You can modify the `Toast` component to match your app's theme and requirements. Customize styles, icons, or actions as needed.

#### Modification Guidelines

- Do not auto-dismiss action-based toasts by default.
- Preserve timer cleanup logic in `useEffect`.
- Avoid mutating the toast array directly.
- Keep `ToastProvider` mounted once at app root.
- Add new toast types only with corresponding UI styles.
