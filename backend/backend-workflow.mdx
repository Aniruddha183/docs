## Phase 2: Development

<Accordion type="single" collapsible>
  <AccordionItem value="development-setup">
    <AccordionTrigger>Setup Local Development Environment</AccordionTrigger>
    <AccordionContent>
      <ul>
        <li>Pull the latest code from the development branch:</li>
        <pre>
          <code>git pull origin development</code>
        </pre>
      </ul>
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="rest-api-development">
    <AccordionTrigger>REST API Development</AccordionTrigger>
    <AccordionContent>
      <ul>
        <li>Create and test the required REST API locally.</li>
        <li>Push your code to a new feature branch:</li>
        <pre>
          <code>
            git checkout -b feature/branchName git add . git commit -m "Commit
            Message" git push origin feature/branchName
          </code>
        </pre>
      </ul>
    </AccordionContent>
  </AccordionItem>
  <AccordionItem value="code-merge-practices">
    <AccordionTrigger>Code Merge Practices</AccordionTrigger>
    <AccordionContent>
      <ul>
        <li>
          Pull updates from the development branch into the feature branch:
        </li>
        <pre>
          <code>git pull origin development</code>
        </pre>
        <li>Ensure only the src directory is added before pushing.</li>
        <li>
          Clear cached files like <code>.env</code>:
        </li>
        <pre>
          <code>git rm -r --cached .env</code>
        </pre>
        <li>
          Create and raise a Pull Request (PR) on GitHub from the feature branch
          to the development branch.
        </li>
      </ul>
    </AccordionContent>
  </AccordionItem>
</Accordion>

## Phase 4: Production Deployment

<Accordion type="single" collapsible>
  <AccordionItem value="production-deployment">
    <AccordionTrigger>Deploy Feature Branch to Production</AccordionTrigger>
    <AccordionContent>
      <ul>
        <li>Push code to a stable feature branch:</li>
        <pre>
          <code>
            git checkout -b feature/branchName git add . git commit -m "Commit
            Message" git push origin feature/branchName
          </code>
        </pre>
        <li>Setup production resources such as S3 buckets and IAM roles.</li>
        <li>Update and test the production environment file.</li>
      </ul>
    </AccordionContent>
  </AccordionItem>
</Accordion>

## Phase 5: Deployment Strategy

<Accordion type="single" collapsible>
  <AccordionItem value="deployment-best-practices">
    <AccordionTrigger>Best Practices</AccordionTrigger>
    <AccordionContent>
      <ul>
        <li>
          Deploy from development to production only when the development branch
          is stable.
        </li>
        <li>
          Limit production releases to critical bug fixes or high-priority
          features.
        </li>
        <li>Ensure a thorough review of changes before deployment.</li>
      </ul>
    </AccordionContent>
  </AccordionItem>
</Accordion>

## Important Notes

<Accordion type="single" collapsible>
  <AccordionItem value="important-notes">
    <AccordionTrigger>Key Considerations</AccordionTrigger>
    <AccordionContent>
      <ul>
        <li>
          Ensure <code>.env</code> files are included in <code>.gitignore</code>
          .
        </li>
        <li>
          When installing packages, check for compact options instead of entire
          packages (e.g., <code>@aws-sdk/client-s3</code>).
        </li>
        <li>
          Maximum Lambda deployment package size is 50 MB (direct upload) or 250
          MB (via S3).
        </li>
      </ul>
    </AccordionContent>
  </AccordionItem>
</Accordion>
