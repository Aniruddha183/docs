## Base URL

```
/api/fitnearn/web/admin/studio
```

## Middleware

### `authenticateAdminToken()`

- **Headers (for user authorization)**:
  -- `genToken` (string): GenToken to verify admin (required)

- **Purpose**: Validates JWT token and authenticates admin user requests.
- **Usage**: Required in all protected admin endpoints.

### `authenticateAdminToken(<permission name)`

- **Purpose**: Validates admin user permission & boundries.
- **Usage**: Required in all protected admin endpoints.

### `idGenerationMiddleware(<prefix>)`

- **Output** : ADMINXXXXX
- **Purpose**: Generates unique query IDs for issue reporting.
- **Usage**: Applied to report creation endpoints.

### `upload.single(<fieldName>)`

- **Headers (for file upload)**:
  - `Authorization` (string): Bearer token (required if uploading file)
  - `x-refresh-token` (string): Refresh token (required if uploading file)
  - `x-username` (string): Username (required if uploading file)
- **Purpose**: Handles file upload using multer to S3
- **Usage**: Required for generating image/attachment URLs for queries

### 6. Add Certificate

Uploads and adds a certificate for the studio owner.

- **Method**: `POST`
- **URL**: `/addCertificate`
- **Middleware**: `authenticateAdminToken`, `idGenerationMiddleware('CERT')`
- **Controller**: `uploadCertificate`
- **Request Body**:
  - `ownerId` (string): Unique studio owner identifier (required)
  - `name` (string): Name of the certificate (required)
  - `provider` (string): Name of the certificate provider (required)
  - `image` (string): URL of the uploaded certificate (required)
  - `key` (string): S3 key for the certificate file (required)
  - `validityFrom` (Date): Date when certificate was issued (required)
  - `validityTo` (Date): Certificate expiry date (required)
  - `studioId` (Datstringe): belonging studioId (required)
- **Success Response**:

```json
{
  "success": true,
  "message": "Certificate uploaded successfully",
  "data": [{...certificateObject}]
}
```

- **Error Responses**:
  - `400 Bad Request`: All fields are required.
  - `404 Not Found`: Studio not found
  - `500 Internal Server Error`

- **Request Body**:

```json
}
```

- **Error Responses**:

### 1. Add Studio Reason / Action Comment

Records administrative actions such as approval, rejection, or review comments for studios, studio owners, or KYC profiles.  
This API is used internally by admins or reviewers to maintain transparency in decision-making and notify the respective studio owners.

- **Method**: `POST`
- **URL**: `/addReason`
- **Controller**: `addReason`
- **Request Body**

```json
{
  "id": "STD00123", //  Required | The identifier of the target entity (studioId, ownerId, or KYC ownerId)
  "type": "approval", // Required | The action type: "rejection", "approval", "review", "payment", "refund", "cancellation", "kycapproval", "kycrejection"
  "reason": "Documents verified", // Optional | Required only for rejection-related actions ("rejection", "kycrejection", "cancellation", "review")
  "adminId": "ADM00045", // Required | The ID of the admin performing the action
  "studioType": "studio", // Required | Target type: "studio" (studio approval/rejection), "owner" (profile approval/rejection), or "kyc" (KYC verification/rejection)
  "isVisible": true // Optional | Determines if the reason/comment is visible to the user; defaults to true for rejection-type actions
}
```

#### Behavior by Type

| Type           | Updates Performed                               | Required `reason`                  | Example Notification |
| -------------- | ----------------------------------------------- | ---------------------------------- | -------------------- |
| `approval`     | Marks studio/owner/KYC as verified/approved     | `"Your studio has been verified!"` |
| `rejection`    | Marks studio/owner as denied and inactive       | `"Your studio got rejected"`       |
| `review`       | Adds internal admin comments for review process | `"Review comment recorded"`        |
| `payment`      | Adds a payment-related comment                  | `"Payment noted"`                  |
| `refund`       | Logs refund comment entry                       | `"Refund processed"`               |
| `cancellation` | Records cancellation-related comments           | `"Booking cancelled"`              |
| `kycapproval`  | Verifies owner’s KYC                            | `"Your KYC is now verified"`       |
| `kycrejection` | Rejects owner’s KYC                             | `"Your KYC has been rejected"`     |

---

#### Success Response

```json
{
  "success": true,
  "message": "approval reason added successfully"
}
```

- **Error Responses**:

- `400 Bad Request` – Missing required fields
- `404 Not Found` – Studio/Owner/KYC not found
- `500 Internal Server Error`
